# DESIGN.md

Welcome to the **Shadcn UI v4 Design System Specification**. This document serves as a persistent, high-fidelity source of truth describing the visual identity, token variables, layout foundations, elevation practices, shape rules, component patterns, and implementation constraints of the Shadcn UI v4 framework. 

This document bridges the gap between human designers and coding agents. It provides a formal definition of every core structural variable in the codebase, alongside the design rationale—the *why*—governing those choices. If you are a developer or an autonomous coding agent, placing this `DESIGN.md` in the root of any compatible project folder should enable you to build a visually perfect, identical, accessible, and responsive user interface conforming exactly to the Shadcn UI v4 visual guidelines.

---

## Overview

Shadcn UI v4 represents a paradigm shift in visual modularity, shifting from a single fixed aesthetic to an **adaptive visual engine** capable of supporting eight distinct aesthetic styles (Vega, Nova, Maia, Lyra, Mira, Luma, Sera, Rhea) layered across two independent component architectures (Radix UI and Base UI) and seven cohesive color palettes.

### Brand Personality

The Shadcn brand is rooted in **meticulous utility, engineered precision, and geometric elegance**. It does not seek to enforce a singular stylistic opinion. Instead, it serves as a blank canvas with highly defined structural boundaries:
- **Utilitarian & Pure:** Information density and readability are prioritized above decorative accents.
- **Engineered & Precise:** Interaction states, boundaries, focus rings, and transition mechanics are mathematically aligned.
- **Adaptive Multiplicity:** By hot-swapping visual layers, the interface can easily morph from an editorial, serif-driven magazine style to a brutalist, sharp monospace terminal, or a fluid, pill-shaped mobile canvas.

### Core Philosophy

1. **Aesthetic Autonomy (Multiplicity):** Styling properties are separated into an isolated registry layer. Designers can switch the "Style" without modifying the functional React code or HTML DOM tree.
2. **Headless & Accessible Foundations:** Radix UI serves as the default behavior and accessibility provider (supporting complete keyboard navigation, aria-roles, focus management). Base UI provides a secondary web-standard base.
3. **No Decorative Bloat:** Avoid heavy drop-shadows, complex 3D gradients, or non-functional border curves unless specifically mandated by a style class. Density and contrast are the primary mechanisms of information hierarchy.

### Style Archetypes (The Eight Visual Paths)

- **Vega (The Default / Familiar):** Clean, neutral, and highly approachable. The standard benchmark of the modern SaaS platform. Balanced paddings, soft `0.625rem` (10px) corners, and high-readability sans-serif fonts.
- **Nova (The Compressed SaaS):** Designed for dashboard layouts where density is paramount. Features reduced internal paddings, small `0.45rem` (7.2px) rounded corners, and compact component heights.
- **Maia (The Organic Round):** Approachable, warm, and spacious. Generous internal spacing, ultra-rounded corners, and soft, friendly sans-serif typefaces.
- **Lyra (The Technical Mono):** Sharp, boxy, and strict. Fully squared corners (`0px` border-radius), high border contrasts, and monospace fonts. Ideal for developer consoles, terminal utilities, and engineering layouts.
- **Mira (The Compact Grid):** Designed for extreme physical data density. Highly constrained spacing, tiny corner offsets, and condensed structures for data-heavy views.
- **Luma (The Luminous Pill):** Soft, fluid, and modern. Employs large rounded pills (`border-radius: 9999px`) for buttons and badges, highly interactive hover offsets, and smooth, playful kinetic transitions.
- **Sera (The Editorial Serif):** Elegant, literary, and classical. Employs serif headings (Playfair Display) alongside high-readability sans-serif body copy, light-background structures, and generous text spacing.
- **Rhea (The Compact Pill):** Blends the fluid, pill-shaped rounded style of Luma with the compressed density of Nova. Highly interactive and space-efficient.

### The Seven Color Palettes

We define seven cohesive color palettes. Each represents a distinct mood and neutral profile:
1. **Neutral:** Pure white to solid black grey. Clean, timeless, completely neutral.

- **Dark Mode Surface Lightening:** In dark mode, instead of adding borders or drop shadows to convey depth, we lighten the base background of containers. Background is `--background` (`oklch(0.145 0 0)`), while cards sitting on top use `--card` (`oklch(0.205 0 0)`), which is visually lighter.
- **Focus Rings:** Ring outlines must use `--ring` with an opacity multiplier to ensure it doesn't wash out.
- **Destructive Highlighting:** Error elements in dark mode use a lighter, more vibrant red (`oklch(0.704)`) to maintain AA contrast ratios against dark backgrounds, whereas light mode utilizes a deeper red (`oklch(0.577)`).

## Typography

The typography system is built as a double-engine model: a flexible, highly legible sans-serif font family for functional operations, navigation, and body content; and a distinct typeface family for headings or monospace environments.

### The Supported Font Families

- **Geist (Modern Sans / Default):** A beautiful, high-readability sans-serif typeface designed with extreme precision for coding interfaces, tables, and functional forms. It provides clean, neutral vertical stems and open apertures.
- **Inter (Familiar Sans):** The legendary web-standard UI font. Exceptional balancing between uppercase letters and numerals.
- **Space Grotesk (Technical Sans):** A geometry-driven typeface featuring wide, crisp proportions, flat-topped counters, and architectural terminal cuts. Extensively used for high-tech labels.
- **Playfair Display (Editorial Serif):** A high-contrast, high-character serif typeface featuring dramatic thick-and-thin stems. Ideal for elegant, classical header blocks.
- **JetBrains Mono (Technical Monospace):** Consistently engineered for developers. Distinct character heights, highly readable vertical columns, and clear glyph separation.

### Typography Hierarchy & Sizing

All typography is divided into twelve distinct levels. For fluid sizing, we prescribe exact line-height multipliers:

1. **Display Large (`display-lg`):** `3.75rem` (60px). Line-height: `1.1`. Weight: `700` (Bold). Letter spacing: `-0.03em`. Reserved for marketing hero headlines.
2. **Display Medium (`display-md`):** `3.00rem` (48px). Line-height: `1.15`. Weight: `700` (Bold). Letter spacing: `-0.02em`.
3. **Heading 1 / Title Large (`title-lg`):** `2.25rem` (36px). Line-height: `1.2`. Weight: `600` (Semi-Bold). Letter spacing: `-0.02em`. Main page titles.
4. **Heading 2 / Title Medium (`title-md`):** `1.875rem` (30px). Line-height: `1.25`. Weight: `600` (Semi-Bold). Letter spacing: `-0.015em`. Secondary section titles.
5. **Heading 3 / Title Small (`title-sm`):** `1.50rem` (24px). Line-height: `1.3`. Weight: `600` (Semi-Bold). Letter spacing: `-0.01em`. Component group cards, table headers.
6. **Heading 4 (`h4`):** `1.25rem` (20px). Line-height: `1.4`. Weight: `600` (Semi-Bold). Letter spacing: `-0.005em`. Card headers, modal headlines.
7. **Body Large (`body-lg`):** `1.125rem` (18px). Line-height: `1.6`. Weight: `400` (Regular). Intro paragraphs, lead copy.
8. **Body Medium (`body-md`):** `1.00rem` (16px). Line-height: `1.5`. Weight: `400` (Regular). Standard body text, descriptions, table cell records.

12. **Code / Monospace (`code-md`):** `0.875rem` (14px). Line-height: `1.5`. Weight: `400` (Regular). Used inside block-codes, code preview snippets, and data cells.

## Layout & Spacing

A strict rhythmic structure is key to Shadcn's visual consistency. Spacing relies on an 8px base unit, scaling mathematically with absolute ratios to enforce visual alignment.

### Grid Rhythm & Margins

- **Base Unit:** 8px. All margins, padding, gaps, and offsets must be multiples of 4px or 8px.
- **Container Layouts:** The core page width is capped at `1400px`. Padding is `1rem` on mobile, scaling to `2rem` on desktop screens.
- **Fixed Wrapper Layouts:** The maximum layout wrapper bounds are locked to `1536px` plus padding on high-resolution displays.

### Layout Utility CSS Classes

1. **`.border-grid`:** A beautiful half-transparent utility class that renders dividers at `--border/50` opacity in light mode, resolving to solid `--border` in dark mode. It structures the page canvas like a drawing blueprint.
2. **`.section-soft`:** Creates a vertical gradient backdrop. It transitions smoothly from the base `--background` canvas to `--surface/40` in light mode, preventing stark background transitions.
3. **`.no-scrollbar`:** Standardizes scrollbar removal across webkit, firefox, and IE/Edge while preserving native mouse and swipe interaction.
4. **`.step`:** Implements auto-incremented step indicators using native CSS counters. Renders steps as round circular badges. On desktop screens, it offsets the indicator into the margin, aligning descriptions to the text gutter.
5. **`.extend-touch-target`:** Improves physical usability on touchscreens. By injecting an absolute pseudo-element (`after:absolute after:-inset-2`), touch interaction targets are expanded by 8px in all directions.

## Elevation & Depth

Visual depth is achieved through **tonal separation** and **rigid boundaries** rather than complex layered shadows. This flat-contrast methodology keeps interfaces sharp and clean.

### Tonal Elevation vs. Dropped Shadows

Instead of layering multiple dropshadows, components rely on background transitions:
- **Base Level (Canvas):** `--background` oklch color.
- **Elevation Level 1 (Cards, Modules, Input Groups):** Sitting directly on `--background`. Employs `--card` or `--surface` colors.
- **Elevation Level 2 (Popover boxes, dropdown menus, context triggers):** Employs `--popover` backdrop with a solid `1px` `--border` outline.

### Modal & Dialog Elevation

For floating modals, Shadcn v4 introduces **Dialog Rings**:
- **`.dialog-ring`:** Modals must not use conventional heavy black shadows. Instead, they use a subtle blur alongside a broad focus ring: `shadow-2xl ring-4 ring-neutral-200/80` (or `ring-neutral-800` in dark mode), giving the floating card a physical outline that separates it from the blurred page canvas.
- **Backdrop Overlays:** Dialog/modal layers use a translucent overlay with a slight blur: `bg-black/10 duration-100 supports-backdrop-filter:backdrop-blur-xs` to isolate the active layout.

## Shapes

Shadcn's shape vocabulary revolves around **curved boundary rules and border-radius relationships**. Corner radius scales dynamically based on the parent component’s volume.

### Dynamically Scaled Radius System

The system defines the primary border-radius variable as `--radius` (locked to `0.625rem` or 10px by default). All auxiliary elements scale relative to this central value:

- **Extra Small Elements (`rounded-xs` /calc(var(--radius) * 0.4)):** `0.25rem` (4px). Applied to checkboxes, toggle stems, and tiny visual indicator blocks.
- **Small Elements (`rounded-sm` / calc(var(--radius) * 0.6)):** `0.375rem` (6px). Applied to buttons, inputs, badge tags, and compact toolbar triggers.
- **Medium Elements (`rounded-md` / calc(var(--radius) * 0.8)):** `0.50rem` (8px). Applied to standard dropdown list triggers, list selections, and cards inside sidebar layouts.
- **Large Elements (`rounded-lg` / var(--radius)):** `0.625rem` (10px). Standard default. Used for main buttons, inputs, alert callouts.
- **Extra Large Elements (`rounded-xl` / calc(var(--radius) * 1.4)):** `0.875rem` (14px). Applied to modal dialog cards, popover menus, and inner card containers.
- **Double Extra Large (`rounded-2xl` / calc(var(--radius) * 1.8)):** `1.125rem` (18px). Applied to outer container cards and dashboard panel layouts.
- **Triple Extra Large (`rounded-3xl` / calc(var(--radius) * 2.2)):** `1.375rem` (22px). Applied to main outer panels in Maia.
- **Infinite Round (`rounded-full`):** `9999px`. Creates perfect visual pill shapes. Applied to avatars, toggle sliders, selection pills, and Luma style components.

### Boundary Lines and Thickness

- **Default Border Width:** Solid `1px`.
- **Active Focus Indicator:** Outer focus rings use `--ring` with `3px` width offsets, leaving a visual gap of `1px` from the component border.

## Do's and Don'ts

To maintain visual integrity and compliance with design system guidelines, ensure you follow these constraints:

### Do's
- **Do Maintain WCAG AA Contrast:** Ensure all typography on backgrounds achieves at least a `4.5:1` contrast ratio. Use the designated OKLCH values.
- **Do Align with Spacing Scales:** Build layouts strictly using multiples of 4px/8px.
- **Do Scale Corners Relative to Volume:** Larger blocks (such as modals) should use larger radius constraints than smaller blocks (such as text fields).
- **Do Utilize Tonal Layers in Dark Mode:** Build visual elevation by shifting towards lighter base backgrounds (`--card` oklch values) instead of casting dark shadows.
- **Do Respect the Aesthetic Archetype:** If you use Lyra (mono/brutalist style), maintain sharp corners (`rounded-none`) across all child elements to preserve consistency.

### Don'ts
- **Don't Mix Stylistic Eras:** Never combine a Luma style capsule button with a Lyra style sharp card.
- **Don't Over-Saturate Backgrounds:** Avoid placing neon colored accents behind primary body text.
- **Don't Invent Arbitrary Corner Radii:** Keep layouts bound to the specified `rounded` scale levels.
- **Don't Bloat CSS with Shadows:** Rely on tonal separators, borders, and dialog focus rings to convey physical depth.
- **Don't Skip Touch Targets:** Avoid setting physical buttons below a 44x44px target on mobile. Use `.extend-touch-target` to increase target areas if necessary.
